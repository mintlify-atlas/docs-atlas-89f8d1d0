## Overview

sptfy.in uses [PocketBase](https://pocketbase.io) for authentication and API access. All authenticated endpoints require a valid PocketBase auth token.

## Authentication Methods

### OAuth2 (GitHub)

The primary authentication method is GitHub OAuth2.

<CodeGroup>
```javascript JavaScript
const pb = new PocketBase('https://your-instance.com');

// Initiate OAuth2 flow
const authData = await pb.collection('users').authWithOAuth2({
  provider: 'github'
});

console.log(authData.token); // JWT token
console.log(authData.record); // User record
```

```python Python
from pocketbase import PocketBase

pb = PocketBase('https://your-instance.com')

# OAuth2 authentication
auth_data = pb.collection('users').auth_with_oauth2(
    provider='github'
)

print(auth_data.token)
```
</CodeGroup>

### Using Auth Tokens

Once authenticated, include the token in subsequent requests:

<CodeGroup>
```bash cURL
curl -X GET "https://api.sptfy.in/api/links" \
  -H "Authorization: Bearer YOUR_AUTH_TOKEN"
```

```javascript JavaScript
const response = await fetch('https://api.sptfy.in/api/links', {
  headers: {
    'Authorization': `Bearer ${pb.authStore.token}`
  }
});
```
</CodeGroup>

## Turnstile Token (Bot Protection)

For unauthenticated link creation, you must include a Cloudflare Turnstile token to prevent abuse.

<ParamField header="X-Turnstile-Token" type="string" required>
  Cloudflare Turnstile challenge token obtained from the client-side widget
</ParamField>

### Getting a Turnstile Token

<Steps>
  <Step title="Add Turnstile Widget">
    Include the Turnstile widget in your HTML:
    ```html
    <div class="cf-turnstile" data-sitekey="YOUR_SITE_KEY"></div>
    <script src="https://challenges.cloudflare.com/turnstile/v0/api.js" async defer></script>
    ```
  </Step>
  
  <Step title="Get Token from Callback">
    Retrieve the token from the Turnstile callback:
    ```javascript
    window.onloadTurnstileCallback = function () {
      turnstile.render('.cf-turnstile', {
        sitekey: 'YOUR_SITE_KEY',
        callback: function(token) {
          // Use this token in X-Turnstile-Token header
          console.log(token);
        }
      });
    };
    ```
  </Step>
  
  <Step title="Include in API Request">
    Pass the token in the `X-Turnstile-Token` header when creating links
  </Step>
</Steps>

### Example with Turnstile

```javascript
const turnstileToken = '...'; // From Turnstile widget callback

const response = await fetch('https://api.sptfy.in/api/links', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'X-Turnstile-Token': turnstileToken
  },
  body: JSON.stringify({
    from: 'https://open.spotify.com/track/...'
  })
});
```

## User Information

### Get Current User

<CodeGroup>
```bash cURL
curl -X GET "https://api.sptfy.in/api/me" \
  -H "Authorization: Bearer YOUR_AUTH_TOKEN"
```

```javascript JavaScript
const response = await fetch('https://api.sptfy.in/api/me', {
  headers: {
    'Authorization': `Bearer ${pb.authStore.token}`
  }
});

const data = await response.json();
console.log(data.user);
```
</CodeGroup>

**Response**

<ResponseField name="user" type="object | null">
  User object if authenticated, null otherwise
  
  <Expandable title="User properties">
    <ResponseField name="id" type="string">
      PocketBase record ID
    </ResponseField>
    
    <ResponseField name="username" type="string">
      GitHub username
    </ResponseField>
    
    <ResponseField name="email" type="string">
      User email address
    </ResponseField>
    
    <ResponseField name="avatar" type="string">
      GitHub avatar URL
    </ResponseField>
    
    <ResponseField name="onboarded" type="boolean">
      Whether user completed onboarding
    </ResponseField>
    
    <ResponseField name="created" type="string">
      ISO 8601 timestamp
    </ResponseField>
  </Expandable>
</ResponseField>

### Update User Profile

Update username and onboarding status.

<CodeGroup>
```bash cURL
curl -X PATCH "https://api.sptfy.in/api/me" \
  -H "Authorization: Bearer YOUR_AUTH_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "username": "newusername"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.sptfy.in/api/me', {
  method: 'PATCH',
  headers: {
    'Authorization': `Bearer ${pb.authStore.token}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    username: 'newusername'
  })
});
```
</CodeGroup>

**Request Body**

<ParamField body="username" type="string" optional>
  New username (3-150 chars, alphanumeric, dots, hyphens, underscores)
</ParamField>

**Response**

<ResponseField name="user" type="object">
  Updated user record
</ResponseField>

**Error Responses**

<ResponseField name="401" type="error">
  Not authenticated
</ResponseField>

<ResponseField name="400" type="error">
  Invalid username format
</ResponseField>

<ResponseField name="409" type="error">
  Username already taken
</ResponseField>

## PocketBase Collections

sptfy.in uses these PocketBase collections:

<Card title="random_short" icon="link">
  Main collection for shortened links. Write operations require authentication or Turnstile token.
</Card>

<Card title="viewList" icon="eye">
  Read-only view of links with user information. Used for fetching user's links.
</Card>

<Card title="analytics" icon="chart-line">
  Analytics data for link visits (country, user agent, timestamps)
</Card>

<Card title="users" icon="user">
  User accounts managed by PocketBase auth
</Card>

## Error Handling

All endpoints return standard HTTP status codes:

| Status Code | Meaning |
|-------------|----------|
| 200 | Success |
| 201 | Created |
| 204 | No Content (successful deletion) |
| 400 | Bad Request (invalid parameters) |
| 401 | Unauthorized (missing or invalid auth token) |
| 403 | Forbidden (not owner of resource) |
| 404 | Not Found |
| 409 | Conflict (duplicate slug) |
| 503 | Service Unavailable (maintenance mode) |

**Error Response Format**

```json
{
  "message": "Error description"
}
```
