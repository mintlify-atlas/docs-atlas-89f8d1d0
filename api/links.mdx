## Overview

The Links API allows you to manage shortened Spotify links. All write operations (create, update, delete) require authentication unless using Turnstile for anonymous link creation.

## Base URL

```
https://api.sptfy.in/api/links
```

## List Links

Retrieve paginated list of links owned by the authenticated user.

<CodeGroup>
```bash cURL
curl -X GET "https://api.sptfy.in/api/links?page=1&perPage=20" \
  -H "Authorization: Bearer YOUR_AUTH_TOKEN"
```

```javascript JavaScript
const response = await fetch('https://api.sptfy.in/api/links?page=1&perPage=20', {
  headers: {
    'Authorization': `Bearer ${pb.authStore.token}`
  }
});

const data = await response.json();
console.log(data.items);
```

```python Python
import requests

response = requests.get(
    'https://api.sptfy.in/api/links',
    params={'page': 1, 'perPage': 20},
    headers={'Authorization': f'Bearer {token}'}
)

data = response.json()
```
</CodeGroup>

### Query Parameters

<ParamField query="page" type="number" default="1">
  Page number for pagination
</ParamField>

<ParamField query="perPage" type="number" default="20">
  Number of items per page (max: 50)
</ParamField>

### Response

<ResponseField name="items" type="array">
  Array of link objects
  
  <Expandable title="Link object properties">
    <ResponseField name="id" type="string">
      PocketBase record ID
    </ResponseField>
    
    <ResponseField name="id_url" type="string">
      Short link slug (e.g., "abc123")
    </ResponseField>
    
    <ResponseField name="from" type="string">
      Original Spotify URL
    </ResponseField>
    
    <ResponseField name="subdomain" type="string">
      Domain used for the short link (e.g., "sptfy.in")
    </ResponseField>
    
    <ResponseField name="utm_view" type="number">
      Number of clicks/visits
    </ResponseField>
    
    <ResponseField name="enable" type="boolean">
      Whether the link is active
    </ResponseField>
    
    <ResponseField name="user" type="string">
      Owner's user ID (empty for anonymous links)
    </ResponseField>
    
    <ResponseField name="created" type="string">
      ISO 8601 timestamp
    </ResponseField>
    
    <ResponseField name="updated" type="string">
      ISO 8601 timestamp
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="page" type="number">
  Current page number
</ResponseField>

<ResponseField name="perPage" type="number">
  Items per page
</ResponseField>

<ResponseField name="totalPages" type="number">
  Total number of pages
</ResponseField>

<ResponseField name="totalItems" type="number">
  Total number of links
</ResponseField>

## Create Link

Create a new shortened link. Requires authentication OR a valid Turnstile token.

<CodeGroup>
```bash cURL (Authenticated)
curl -X POST "https://api.sptfy.in/api/links" \
  -H "Authorization: Bearer YOUR_AUTH_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "from": "https://open.spotify.com/track/3n3Ppam7vgaVa1iaRUc9Lp",
    "slug": "mytrack"
  }'
```

```bash cURL (Anonymous with Turnstile)
curl -X POST "https://api.sptfy.in/api/links" \
  -H "Content-Type: application/json" \
  -H "X-Turnstile-Token: YOUR_TURNSTILE_TOKEN" \
  -d '{
    "from": "https://open.spotify.com/track/3n3Ppam7vgaVa1iaRUc9Lp"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.sptfy.in/api/links', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${pb.authStore.token}`
  },
  body: JSON.stringify({
    from: 'https://open.spotify.com/track/3n3Ppam7vgaVa1iaRUc9Lp',
    slug: 'mytrack', // optional
    subdomain: 'sptfy.in' // optional
  })
});

const link = await response.json();
console.log(`Created: https://${link.subdomain}/${link.id_url}`);
```

```python Python
import requests

response = requests.post(
    'https://api.sptfy.in/api/links',
    headers={
        'Authorization': f'Bearer {token}',
        'Content-Type': 'application/json'
    },
    json={
        'from': 'https://open.spotify.com/track/3n3Ppam7vgaVa1iaRUc9Lp',
        'slug': 'mytrack'
    }
)

link = response.json()
```
</CodeGroup>

### Request Body

<ParamField body="from" type="string" required>
  Original Spotify URL to shorten (must be a valid URL)
</ParamField>

<ParamField body="slug" type="string" optional>
  Custom slug for the short link. If not provided, a random slug will be generated (4-8 characters, alphanumeric).
  
  **Constraints:**
  - Must be unique
  - Cannot be a reserved word (e.g., "api", "admin", "dash")
  - Alphanumeric characters only
</ParamField>

<ParamField body="subdomain" type="string" optional default="sptfy.in">
  Subdomain to use for the short link
</ParamField>

<ParamField body="turnstileToken" type="string" optional>
  Cloudflare Turnstile token (required if not authenticated)
</ParamField>

### Response

Returns the created link object with status code `201 Created`.

```json
{
  "id": "new_record_id",
  "id_url": "mytrack",
  "from": "https://open.spotify.com/track/3n3Ppam7vgaVa1iaRUc9Lp",
  "subdomain": "sptfy.in",
  "utm_view": 0,
  "enable": true,
  "user": "user_id_here",
  "created": "2024-01-15T10:30:00.000Z",
  "updated": "2024-01-15T10:30:00.000Z"
}
```

### Error Responses

<ResponseField name="400" type="error">
  Bad request - missing `from` parameter or invalid data
  ```json
  {"message": "from is required"}
  ```
</ResponseField>

<ResponseField name="409" type="error">
  Conflict - slug already taken
  ```json
  {"message": "slug taken"}
  ```
</ResponseField>

<ResponseField name="503" type="error">
  Service unavailable - maintenance mode active
  ```json
  {"message": "Service temporarily unavailable for maintenance"}
  ```
</ResponseField>

## Update Link

Update an existing link's destination URL, slug, or subdomain. Requires authentication and ownership.

<CodeGroup>
```bash cURL
curl -X PATCH "https://api.sptfy.in/api/links/RECORD_ID" \
  -H "Authorization: Bearer YOUR_AUTH_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "from": "https://open.spotify.com/track/NEW_TRACK_ID",
    "id_url": "newslug"
  }'
```

```javascript JavaScript
const response = await fetch(`https://api.sptfy.in/api/links/${recordId}`, {
  method: 'PATCH',
  headers: {
    'Authorization': `Bearer ${pb.authStore.token}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    from: 'https://open.spotify.com/track/NEW_TRACK_ID',
    id_url: 'newslug'
  })
});

const updated = await response.json();
```
</CodeGroup>

### URL Parameters

<ParamField path="id" type="string" required>
  PocketBase record ID of the link to update
</ParamField>

### Request Body

All fields are optional - only include fields you want to update.

<ParamField body="from" type="string" optional>
  New destination URL
</ParamField>

<ParamField body="id_url" type="string" optional>
  New slug (must be unique)
</ParamField>

<ParamField body="subdomain" type="string" optional>
  New subdomain
</ParamField>

### Response

Returns the updated link object.

```json
{
  "id": "record_id",
  "id_url": "newslug",
  "from": "https://open.spotify.com/track/NEW_TRACK_ID",
  "subdomain": "sptfy.in",
  "utm_view": 42,
  "enable": true,
  "user": "user_id_here",
  "created": "2024-01-15T10:30:00.000Z",
  "updated": "2024-01-15T14:22:00.000Z"
}
```

### Error Responses

<ResponseField name="401" type="error">
  Not authenticated
</ResponseField>

<ResponseField name="403" type="error">
  Forbidden - you don't own this link
</ResponseField>

<ResponseField name="404" type="error">
  Link not found
  ```json
  {"message": "not found"}
  ```
</ResponseField>

<ResponseField name="409" type="error">
  Conflict - new slug already taken
  ```json
  {"message": "slug taken"}
  ```
</ResponseField>

<ResponseField name="400" type="error">
  Update failed
  ```json
  {"message": "update failed"}
  ```
</ResponseField>

## Delete Link

Delete a link. Requires authentication and ownership.

<CodeGroup>
```bash cURL
curl -X DELETE "https://api.sptfy.in/api/links/RECORD_ID" \
  -H "Authorization: Bearer YOUR_AUTH_TOKEN"
```

```javascript JavaScript
const response = await fetch(`https://api.sptfy.in/api/links/${recordId}`, {
  method: 'DELETE',
  headers: {
    'Authorization': `Bearer ${pb.authStore.token}`
  }
});

if (response.status === 204) {
  console.log('Link deleted successfully');
}
```

```python Python
import requests

response = requests.delete(
    f'https://api.sptfy.in/api/links/{record_id}',
    headers={'Authorization': f'Bearer {token}'}
)

if response.status_code == 204:
    print('Deleted')
```
</CodeGroup>

### URL Parameters

<ParamField path="id" type="string" required>
  PocketBase record ID of the link to delete
</ParamField>

### Response

Returns status code `204 No Content` on success (empty body).

### Error Responses

<ResponseField name="401" type="error">
  Authentication required
  ```json
  {"message": "Authentication required"}
  ```
</ResponseField>

<ResponseField name="403" type="error">
  Not authorized to delete this link
  ```json
  {"message": "Not authorized to delete this link"}
  ```
</ResponseField>

<ResponseField name="404" type="error">
  Link not found
  ```json
  {"message": "Link not found"}
  ```
</ResponseField>

<ResponseField name="500" type="error">
  Server error during deletion
  ```json
  {"message": "Failed to delete link"}
  ```
</ResponseField>

## Bulk Delete Links

Delete multiple links in a single request. Maximum 5 links per request (free tier limit).

<CodeGroup>
```bash cURL
curl -X DELETE "https://api.sptfy.in/api/links/bulk" \
  -H "Authorization: Bearer YOUR_AUTH_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "ids": ["record_id_1", "record_id_2", "record_id_3"]
  }'
```

```javascript JavaScript
const response = await fetch('https://api.sptfy.in/api/links/bulk', {
  method: 'DELETE',
  headers: {
    'Authorization': `Bearer ${pb.authStore.token}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    ids: ['record_id_1', 'record_id_2', 'record_id_3']
  })
});

const result = await response.json();
console.log(`Deleted: ${result.deleted.length}, Failed: ${result.failed.length}`);
```
</CodeGroup>

### Request Body

<ParamField body="ids" type="array" required>
  Array of record IDs to delete (max 5 items)
</ParamField>

### Response

Returns a summary of the deletion operation.

```json
{
  "deleted": ["record_id_1", "record_id_2"],
  "failed": [
    {
      "id": "record_id_3",
      "reason": "Not authorized"
    }
  ]
}
```

<ResponseField name="deleted" type="array">
  Array of successfully deleted record IDs
</ResponseField>

<ResponseField name="failed" type="array">
  Array of failed deletions with reasons
  
  <Expandable title="Failed item properties">
    <ResponseField name="id" type="string">
      Record ID that failed to delete
    </ResponseField>
    
    <ResponseField name="reason" type="string">
      Reason for failure (e.g., "Not authorized", "Delete failed")
    </ResponseField>
  </Expandable>
</ResponseField>

### Error Responses

<ResponseField name="401" type="error">
  Authentication required
  ```json
  {"message": "Authentication required"}
  ```
</ResponseField>

<ResponseField name="400" type="error">
  Invalid request - missing/invalid ids array or exceeds limit
  ```json
  {"message": "Cannot delete more than 5 links at once (free tier limit)"}
  ```
</ResponseField>

## Expand Spotify URL

Expand shortened Spotify URLs (spotify.link, spotify.app.link) to their final open.spotify.com destination.

<CodeGroup>
```bash cURL
curl -X POST "https://api.sptfy.in/api/expand-url" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://spotify.link/abc123"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.sptfy.in/api/expand-url', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    url: 'https://spotify.link/abc123'
  })
});

const data = await response.json();
console.log(data.expanded);
```
</CodeGroup>

### Request Body

<ParamField body="url" type="string" required>
  Shortened Spotify URL to expand (must be from spotify.link or spotify.app.link)
</ParamField>

### Response

```json
{
  "expanded": "https://open.spotify.com/track/3n3Ppam7vgaVa1iaRUc9Lp",
  "original": "https://spotify.link/abc123",
  "wasExpanded": true
}
```

<ResponseField name="expanded" type="string">
  Final expanded URL (open.spotify.com)
</ResponseField>

<ResponseField name="original" type="string">
  Original URL that was provided
</ResponseField>

<ResponseField name="wasExpanded" type="boolean">
  Whether expansion was performed (false if already a Spotify URL)
</ResponseField>

### Error Responses

<ResponseField name="400" type="error">
  Invalid URL or unsupported domain
  ```json
  {"message": "URL is not from a supported Spotify domain"}
  ```
</ResponseField>

<ResponseField name="500" type="error">
  Failed to expand URL
  ```json
  {"message": "Failed to expand URL: timeout"}
  ```
</ResponseField>

## Redirect Endpoint

Access a shortened link and redirect to the original URL.

### Endpoint

```
GET /:slug
```

<CodeGroup>
```bash cURL
curl -L "https://sptfy.in/abc123"
```

```javascript JavaScript
// Browser automatically follows redirects
window.location.href = 'https://sptfy.in/abc123';
```
</CodeGroup>

### URL Parameters

<ParamField path="slug" type="string" required>
  Short link slug (e.g., "abc123")
</ParamField>

### Behavior

<Steps>
  <Step title="Lookup">
    System looks up the slug in the `viewList` collection
  </Step>
  
  <Step title="Bot Detection">
    Checks if the user agent is a bot/crawler (WhatsApp, Telegram, Twitter, etc.)
  </Step>
  
  <Step title="Analytics Tracking">
    If not a bot, creates an analytics record with:
    - User agent
    - Country (from CF-IPCountry header)
    - Timestamp
    - Link ID
  </Step>
  
  <Step title="View Count Increment">
    If not a bot, increments the `utm_view` counter in the `random_short` collection
  </Step>
  
  <Step title="Redirect">
    Issues a 301 redirect to the original URL
  </Step>
</Steps>

### Response

HTTP 301 redirect to the original Spotify URL.

**Headers:**
```
Location: https://open.spotify.com/track/...
Status: 301 Moved Permanently
```

### Error Responses

<ResponseField name="404" type="error">
  Link not found
  ```html
  Link does not exist, but may be available in the future.
  yeehaw 🔍🤠
  ```
</ResponseField>

### Bot Detection

The following user agents are detected as bots and will NOT trigger analytics or view count increments:

- whatsapp
- telegram  
- twitter
- bot, crawler, spider
- facebookexternalhit
- google-safety, google-firebase
- okhttp, axios, node-fetch
- python-requests
- And more...

<Note>
Bot requests are logged but do not affect analytics to prevent inflated statistics.
</Note>
